# FinTech SQL Analytics Project

This project analyzes fintech transaction data using PostgreSQL and SQL to uncover insights related to transaction behavior, payment patterns, customer activity, and risk signals.

## Project Goal
To build a business-oriented SQL analytics project in the fintech domain using relational database design, data import workflows, and analytical SQL queries.

## Project Evolution
### Phase 1
Built the relational schema and practiced core SQL concepts such as filtering, CASE WHEN, and JOIN operations.

### Phase 2
Expanded the project with a larger transaction dataset and business-focused analyses such as fraud-risk exploration, payment behavior analysis, and customer activity insights.

## Tech Stack
- PostgreSQL
- SQL
- DBeaver
- Git & GitHub

## Project Structure
- `data/` → raw and processed datasets
- `sql/` → schema, import, cleaning, and analysis scripts
- `docs/` → documentation, analysis questions, and final insights
- `images/` → ER diagram and screenshots

## Key Analysis Areas
- transaction analysis
- payment method analysis
- customer behavior analysis
- fraud-risk exploration
- business insights

## Status
Project is being expanded from foundational SQL work into a portfolio-ready fintech analytics case study.
