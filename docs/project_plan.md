# Project Plan

## Project Overview
This project analyzes transaction data in a fintech context to uncover insights about customer behavior, transaction patterns, merchant performance, and payment success rates.

## Business Goal
The goal of this project is to simulate a real-world data analysis scenario in a fintech company.  
By working with relational data and SQL, the project aims to generate insights that support business decisions such as customer engagement, transaction efficiency, and risk monitoring.

## Main Entities
The analysis is based on the following core entities:

- customers
- accounts
- merchants
- transactions

## Key Business Questions
The analysis focuses on answering the following business questions:

1. Who are the most active customers?
2. Which customers generate the highest transaction volume?
3. Which merchants receive the most transactions?
4. Which merchant categories generate the highest total amount?
5. What are the most commonly used payment methods?
6. Which payment methods have the highest success rate?
7. What is the failed transaction rate?
8. Which customers appear to be inactive recently?
9. Are there merchants with unusually high failed transaction rates?
10. Are there customers with unusually high transaction activity in short periods?

## SQL Topics Covered
The project includes the following SQL concepts:

- SELECT
- WHERE
- ORDER BY
- DISTINCT
- CASE WHEN
- GROUP BY
- HAVING
- INNER JOIN
- LEFT JOIN
- Subqueries
- CTEs
- Window Functions
- Date-based Analysis

## Expected Outputs
The project will produce:

- a structured PostgreSQL schema
- organized SQL query scripts
- business-oriented analytical outputs
- clear documentation of findings
