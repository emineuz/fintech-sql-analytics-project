# Project Evolution

## Phase 1 — SQL Foundations
In the first phase of the project, the focus was on learning and applying core SQL concepts using a structured fintech schema.  
This phase included:

- relational schema design
- primary key and foreign key relationships
- basic filtering queries
- CASE WHEN logic
- JOIN operations

## Phase 2 — Real Dataset Upgrade
In the second phase, the project is being expanded into a portfolio-ready fintech analytics case study using a larger dataset and business-oriented analytical questions.

The focus of this phase includes:

- importing a larger transaction dataset into PostgreSQL
- data quality checks and cleaning
- fraud-risk exploration
- customer behavior analysis
- business insight generation
